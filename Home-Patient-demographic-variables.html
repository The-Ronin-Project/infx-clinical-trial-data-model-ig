Patient demographic data

The CTDM does not contain a Patient profile, but rather transforms the demographic variables that are typically contained in the FHIR Patient resource into individual observations. The purpose of this transformation is to enable the request and return of only the specific set of demographic variables included in a given clinical trial.

Following is a list of the demographic variables contained in the FHIR R4 Patient resource and their coded representations as observations.

extension.race* http://snomed.info/sct 103579009 Race (observable entity)

*Note: the Race and Ethnicity elements of the FHIR Patient profile each contain three different sub-elements for reporting race and ethnicity data in different formats. The first is the OMB category, the second is the Detailed CDC category, and the third is free text. Only the free text value is required, but all three may be reported for any given patient.

When race and ethnicity are transformed into observations, there will be as many observations as there are populated sub-elements. Each one will be distinguished based on the note.text as folows:

  • ethnicity.ombCategory: note.text = OMB Ethnicity Category;
  • ethnicity.detailed: note.text = Detailed CDC Ethnicity;
  • ethnicity.text: note.text = Free-text Ethnicity value;
  • race.ombCategory: note.text = OMB Race Category;
  • race.detailed: note.text = Detailed CDC Race;
  • race.text: note.text = Free-text Race value
