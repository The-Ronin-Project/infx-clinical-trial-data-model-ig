CTDM Observation profile

The CTDM Observation profile is a simplified version of the Ronin Common Data Model (RCDM) Observation profile. EHR- and Ronin-specific identifier and reference elements have been removed, and extensions for the study subject identifier and timestamp of data transform have been added.

The meta tag element of the CTDM Observation profile contains the Clinical Trials data dictionary variable that a given observation belongs to, as appropriate. For example, an observation containing a patient's sex will have a tag.display value of 'Sex', and a serum sodium observation will have a tag.display value of 'Sodium'. A given observation may have more than one tag.

The cardinality, or whether each element is optional, required, and how many instances are allowed, is provided in shorthand in the third column. The first number indicates the minimum number of times an element must be present, and the second number indicates the maximum number.

Examples:

  • 0..1: the element must be present at least 0 times, and may be present a maximum of once. Essentially, this means that element is optional and if present, may only have one value.
  • 0..*: the element must be present at least 0 times, and may be present infinite times. This means that the element is optional and if present, may have as many values as needed.
  • 1..1: the element must be present once and only once. This means the element is required and may only have once value.

Note: the meta.tag element has a cardinality of 0..*, meaning that it is not required but if present, may have multiple values (i.e., a single observation may have 0 to many tags). However, for each instance of meta.tag, the system, version, and display are all 1..1, meaning for each tag, all three elements are required and must have a single value.
